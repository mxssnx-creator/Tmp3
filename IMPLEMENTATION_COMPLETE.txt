ARCHITECTURE USED
-----------------
- AlertManager (lib/alerting-system.ts)
  - Multi-channel support (Slack, webhook, email, PagerDuty)
  - Built-in deduplication (1-minute window)
  - Severity-based filtering
  
- REST API (app/api/alerts/route.ts)
  - POST /api/alerts - Send alerts
  - GET /api/alerts - View history
  - DELETE /api/alerts - Clear history

- Error Handler Integration (lib/error-handler.ts)
  - Automatic critical error → alert conversion
  - Seamless with existing application errors

CONFIGURATION
-------------
1. Copy .env.example to .env.local
2. Add Slack webhook URL:
   SLACK_WEBHOOK_URL=https://hooks.slack.com/services/YOUR/WEBHOOK/URL

3. Configure alert manager (code or API):
   - Enable webhook channel
   - Set minimum severity level
   - Configure deduplication window

USAGE EXAMPLES
--------------
API Call:
  curl -X POST http://localhost:3001/api/alerts \
    -H "Content-Type: application/json" \
    -d '{"title":"Alert","message":"Test","severity":"warning","source":"test"}'

Programmatic:
  import { alertManager, AlertSeverity } from '@/lib/alerting-system'
  await alertManager.sendAlert('Title','Message',{severity: AlertSeverity.WARNING})
